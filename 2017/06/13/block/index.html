block 详解

发表于 2017-06-13

block

  • block作为C语言的扩展,和其他语言的闭包是一回事,block中需要自己管理内存,Block允许开发者在两个对象之间将任意的语句当做数据进行传递.block的使用很像函数指针,不过与函数最大的不同是block可以访问函数外部变量的值,换句话说,block不仅实现函数的功能,还能携带函数的执行环境;

  • block其实包含两部分的内容:

    1. block执行的代码,这是在编译的时候已经生成好的.
    2. 一个包含block执行时所需要的所有外部变量的数据结构.block将使用到的变量的值建立一份快照拷贝到栈上.

block 中修饰变量的__block 和 static

1. 局部变量

  • 以const变量被拷贝并存储到block中，也就是说block对其是只读的。如果尝试在block内部给make变量赋值，会抛出编译器错误。
  • 以const拷贝的方式访问非局部变量，意味着block实际上并不是真正的访问了非局部变量——只不过在block中创建了非局部变量的一个快照。当定义block时，无论非局部变量的值是什么，都将被冻结，并且block会一直使用这个值，即使在之后的代码中修改了非局部变量的值。

2. static修饰符的全局变量

  • 因为全局变量或者静态变量在内存中的地址是固定的,block在读取该变量值的时候,直接冲其内部读取,获得新的值,而不是在定义是copy的常量

3. __block修饰的对象

  • 这将告诉block对非局部变量做引用处理，在block外部make变量和内部的make变量创建一个直接的链接(direct link)。现在就可以在block外部修改make，然后反应到block内部，反过来，也是一样。
  • 对于用__block修饰的外部变量引用，block是复制其引用地址来实现访问的

block 的类型和内存管理

根据block在内存中的位置分为三种类型:NSGlobalBlock,NSStackBlock,NSMallocBlock.

NSGlobalBlock : 与全局变量一样，block设置在程序的数据区域（.data区域中）

NSStackBlock : block设置在栈上

NSMallocBlock : block设置在堆上

  1. 配置全局变量的block,从变量作用域外也可以通过指针安全的使用;
  2. 设置在栈上的block,如果其所属的变量作用域结束,block就会作废;
  3. 由于__block变量也是配置在栈上,所以同样地,如果所属的作用域结束,则该_block变量也会被废弃,所以将_block和block变量从栈上复制到堆上解决问题.

注:另外，对于已经在堆区以及配置在程序的数据区域的block，调用了copy方法，会出现

由此可以得出，可以对 array 进行添加操作，因为 array 对象是存在于堆上！可以对堆上的对象进行操作，所以操作的对象一直是初始创建的 array。

注：如果对 array 在 block 中 重新创建一个array = [NSMutableArray array];会报错，因为你将又重新开辟了一段内存空间，企图更改栈中 array的首地址，所以报错。
